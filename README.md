# Podcast Pipeline

An end-to-end automation toolkit that picks up where your recording leaves off — normalizing audio, generating metadata, building Jekyll pages, uploading to platforms, and rendering video.

---

## Features

- **Audio normalization** — levels and loudness targets applied consistently across episodes
- **Metadata generation** — titles, descriptions, tags, and chapter markers created automatically
- **Jekyll page generation** — episode pages scaffolded and ready to publish
- **Platform upload** — pushes finished episodes to podcast hosts and distribution platforms
- **Video generation** — produces video versions of episodes for YouTube and similar platforms

- Python 3.9+

Install Python dependencies:

## Configuration

Copy the example config and fill in your values:

```bash
cp config.example.yml config.yml
```

Key settings in `config.yml`:

```yaml
audio:
  target_lufs: -16
  sample_rate: 44100

jekyll:
  output_dir: ../_posts   # Path to your Jekyll site's posts directory

platforms:
  podbean:
    api_key: YOUR_KEY
  youtube:
    client_secrets: credentials/youtube.json

video:
  resolution: 1920x1080
  background: assets/background.png
```

## Usage

### Run the full pipeline

```bash
python run.py --episode path/to/episode.wav
```

This will execute all stages in sequence: normalize → metadata → Jekyll page → upload → video.

### Run individual stages

```bash
# Normalize audio only
python audio/normalize.py path/to/episode.wav

# Generate metadata only
python metadata/generate.py path/to/episode.wav

# Build Jekyll page only
python jekyll/build_page.py --meta path/to/episode.json

# Upload audio to platforms
python upload/upload_audio.py path/to/episode.mp3

# Render video
python video/render.py path/to/episode.mp3

# Upload video
python upload/upload_video.py path/to/episode.mp4
```

### Options

| Flag | Description |
|---|---|
| `--episode` | Path to the source audio file |
| `--skip-upload` | Run all stages except uploading |
| `--skip-video` | Skip video generation and upload |
| `--dry-run` | Process everything but don't write files or upload |
| `--config` | Path to a custom config file (default: `config.yml`) |
